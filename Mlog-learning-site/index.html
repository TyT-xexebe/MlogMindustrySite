Some basic about processor


  1. A processors start at line 0 (line is used instead of instruction)

  2. Scheme with all block and floor @index ( without erekir ) download

  3. A site for mindustry codding

  4. M-log special @ variables:

    • @thisx / @thisy
      // cooordinated x / y of this proccesor

    • @this
      // object of proccesor

    • @time
      // a time what you spend in world in milliseconds ( when you create the world and join in it the @time automaticaly start counting, when you leave or pause game counting stopped , on servers @time starts when world load )

    • @tick
      // @tick do a simple what do a @time, but in ticks ( A tick is similar to FPS for most purposes )

    • @unit
      // variable to access a unit returned from using 'bind'. Can be used as object to sensor data from the unit.

    • @counter
      // indicates the line of code currently being performed. It's a very interesting variable, explained further here.

    • @mapw / @maph
      // map weight / height in tiles

    • @links
      // total number of blocks linked (connected) to the processor. To link a block, select the processor and then select a different block. It will have a purple diamond and a name to indicate that it is linked.

    • @ipt
      // instruction per tick (number of lines of code performed each tick. A tick is similar to FPS for most purposes)

  5. When code passes the last line, it will start again from line 0

  6. Each processor type has different processing speeds (you can read this from @ipt): micro processor(2ipt or 120 lines/sec at 60 FPS) , logic processor (480 / sec) , hyper proccesor ( 1500 / sec )

  7. A memory cell can store a limited amount of numbers (double or float32). A memory cell stores 64 numbers, a memory bank 512 numbers.

  8. All commands and words are case sensitive e.g. '@firstItem ', not '@firstitem '

  9. Most comparisons are coercive - 'true' and 'false' will also be equal to 1 and 0 respectively. Be careful, null is also coerced to 0. To avoid this, use the strict equals comparison, ===.

  10. If you write code outside Mindustry (e.g. on site or in VScode ) you can use jump labels (absolute) instead of jump numbers (relative) to avoid maintaining jumps

Input and Output

We have 4 commands of this category: read , write , draw , print



Command ' read '
return a variable from mem-cell block

syntax of read: read var cell1 num

' var ' is the name of the variable to store the output from the cell, 'cell1' is a linked mem-cell (although it can be linked in other ways), ' num ' is a 0-based index in mem-cell where to read the value from.

Example:


read aCountOFunits cell1 0
// so we created a variable 'aCountOfUnits' then we tell the processor from which cell to find a value, and index of the cell for the specific number.


Command ' write '
write a any variable to mem-cell, but it can write only numbers, variable values ( in number or 1 / 0 ( true / false )!

syntax of write: write var cell1 num

' var ' is the name of the variable where a number is stored. It can also be a static number. ' cell1 ' its a linked mem-cell, ' num ' is a 0-based index in mem-cell where to write the value.

Example:


write aCountOFunits cell1 0
// so we write a variable 'aCountOfUnits' at index 0 in mem-cell

Now lets connect a read and write commands in code!

proccessor 1:
set aCountOFunits 5
write aCountOFunits cell1 3
 proccessor 2:
read count cell1 3

Processor 1 sets a value to 'aCountOfUnits' variable and then stores this to mem-cell at index 3. Processor 2 reads a 'aCountOfUnits' at index 3, but stores it a variable named 'count'.



Command ' draw ' draw lines, shapes and images to display in a logic display. This is a detailed topic and will be completed at a later date.

syntax's of draw:

  • draw clear 0 0 0
  • draw color 0 0 0 0
  • draw col 0
  • draw stroke 0
  • draw line 0 0 0 0
  • draw rect 0 0 0 0
  • draw lineRect 0 0 0 0
  • draw poly 0 0 0 0 0
  • draw triangle 0 0 0 0 0 0
  • draw image 0 0 0

Extend of 0 you need write a name's / coordinates / icons and any information for draw any what you want.



Command ' print '
stores (appends) a string to be sent to a message block. You need to use print flush to send the stored print string to a message block.

A syntax of print: print ""

for write text you need add any text in " " , if you want write a value of any variable you need write it name and without "

Example:


print "count"
// will write a text in " " ( count )
print count
// will write a value of variable " count "

print "\n"
// will print some text from new line
print "[red]red , [#669980]some color , []white"
// you can print colored text by simple names of color [red] or by #hex colors, [] deleting color
// you can print icons from the game by shift-clicking the icon in the core-database to copy it.

Command ' getlink'
need to linked to blocks what you link to processor

syntax of getlink: getlink link num

' link ' is a variable, ' num ' is index of the blocks linked to the processor in the order they are linked at the start of the code loop, num can be from 0 to the maximum number of blocks in the processors range.
For example we have processor, and it linked to 5 blocks, and we need to know what block is it

Example:


getlink linked 0
print linked
printflush message1

this code will be check the 1st block linked to the processor and print it to a message block ( coerced to block type ). If, instead, we enter 3 for num, the linked block returned will be the 4th linked block. The returned variable is a block object and is extremely useful



Command ' control '
controls blocks available settings. Typically, enabled / disabled ( off / on ), configuration, shoot and shootp from turret and color.

A syntax's of control:

  • control enabled block1 0 // enabling a block
  • control shoot block1 0 0 0 0 // control a turret to shoot to coordinates
  • control shootp block1 0 0 // control a turret toshoot at enemy in it radius
  • control config block1 0 // configurating blocks to something
  • control color block1 0 // setting color for illyminator?.. very useless..


control enabled block slct
' block ' is a block object like reactor1 or returned getlink variable, ' slct ' you write 0 / 1, true / false. ' false ' or 0 will disable this block (if you have block status enabled in settings the diamond will turn solid purple), ' true ' or 1 will enable this block.
Example:

getlink link 2
control enabled link 0
 // this code will disable the 3rd block linked to the processor

control shoot block x y sh
' block ' is a block object like turret1 or returned getlink variable, ' x ' and ' y ' are coordinates where the block should shoot, ' sh ' is the order to shoot or not shoot, set 1 to shoot and 0 to stop shooting. This is remembered and must shooting must be turned off manually
Example:

control shoot foreshadow1 100 100 1
// foreshadow1, the turret linked to the processor, will shoot to coordinates 100,100. Now i will be show a harder example:
getlink block 1
sensor x @x block
sensor y @y block
control shoot hail1 x y 1
 // Yeah, we haven't learned the 'sensor' command yet, but these 2 commands simply check the x and y coordinates in the block returned by getlink. Important, first we need to link the processor to a hail and then to link any other block. getlink returns the second block, then sensor returns the coordinates and hail will shoot at the block!

control shootp block unit sh
You already know what ' block ' and ' sh ' mean from the shoot command. ' unit ' is a unit object returned from the radar / uradar commands. This is new, so I will show an example with it
Example:

getlink link 0
radar enemy any any distance link 1 varE
control shootp link varE 1
 // Yeah maybe you don't yet understand what radar does, but simply: it searches for a unit that is a combination of enemy / ally / player and any any any around the turret and returns the unit object as a variable if it found, so we can shoot at the unit from the linked turret.

control config block type
' type ' is to what you want configure the block to, e.g. @copper for sorters / unloaders or @flare for an air-factory.
Example:

control config sorter1 @lead
// configure a linked sorter to lead
control config factory1 @mono
// configure a linked air factory to mono

control color illuminator1 num
Seriously, idk why you might want to use this to alter illuminators. It's a pretty pointless command. You link the illuminator and write any number and it will display some color (weakly, best seen at night time). Or you can reduce the number of commands and use packcolor result 0 0 0 0 . Color is input as four different channels RGB and Alpha with values from 0-1. Colors can also be input as RGB hex.



Command ' radar '
search for units around turret. There are many inputs, good luck understanding it!

radar enemy any any distance turret 1 result
syntax of selecting unit 'enemy any any' allows up to 3 filters to narrow search results

  • any // any unit
  • enemy // any enemy unit
  • ally // any ally unit
  • player // player
  • attacker // any unit with weapon
  • flying // any flying unit
  • ground // any ground unit
  • boss // any unit with boss effect

You can select 3 filters to narrow the search for the unit you want what unit you want. For example: enemy flying any will be search for an enemy flying units


Also we have 5 sorting options:

  • distance // distance to turret
  • shield // unit with shield
  • health // unit HP
  • maxHealth // unit maxHP
  • armor // unit with armor

Now I'll explain health and max Health, for example we have 2 unit in turret radius a quasar with 150HP ( a full HP quasar is 640 ) and mega with 400 HP ( a full HP of mega: 460 ) if we select a ' maxHealth ' a turret will be looking at quasar, if ' health ' turret will be loking at mega, because mega have more HP then quasar at this moment, but a full HP more in quasar


A number in ' radar enemy any any distance turret 1 result ' is sort order of the sorting option. Selecting unit, for example we also have two units, turret search by distance, a quasar closer to the turret than a mega, so turret return the quasar, however, if you write 2 a turret will return the mega. You cannot search units in the middle of the sort options.

OK, now we can try some code with this command:

radar enemy flying any maxHealth hail1 1 Result
print Result
printflush message1

This code will display the name of the enemy flying unit with maxHealth within the range of hail1 to message1

Command 'ubind'

A syntax of ubindubind @unit

If write code like This ubind @flare

Command ' ucontrol '
ontrol units. This will set the processor to the unit controller. Be careful not to control units already controlled by other processors.
Syntaxs of ucontrol:


  • ucontrol idle // just don't do anything
  • ucontrol stop // stops all unit actions.
  • ucontrol move x y // unit move to ' x ' ' y ' coordinates
  • ucontrol approach x y r // unit move to ' x ' ' y ' coordinates but around it radius( r )
  • ucontrol boost x // boosting for nova tree, ( 0 / 1 )
  • ucontrol target x y s // shoot on ' x ' ' y ' coordinates ' s ' = shoots ( 1 / 0 )
  • ucontrol targetp u s // ' u ' = unit ( can be search by radar or uradar )' s ' = shoots ( 1 / 0 ). Accounts for projectile movement ( better aim )
  • ucontrol itemDrop to amount // ' to ' = what block/ core / some container1 etc, ' amount ' of resources
  • ucontrol itemTake from type amount // taking ' type ' of resource ( @copper ) from any ' block ' and ' amount '
  • ucontrol payDrop // dropped units in unit ( mega - oct )
  • ucontrol payTake @unit / @block // take a unit or block ( first need go to coordinates of this unit / block, then use this command with 0 for blocks and 1 for units ). Check size of unit / block and carrier capacity. for pay take a block need write @nameOfBLock1 , without 1 this command will be don't work!
  • ucontrol payEnter // enter this unit in payload conveyor or reconstructor ( first go to the coordinate of this reconstructor / payload conveyor then check if unit within this coordinates if yes = ucontrol payEnter if you dont do this unit will be can enter in another reconstructor or payload conveyor )
  • ucontrol mine x y // mine ore at coordinates
  • ucontrol flag value // flagging unit using numbers
  • ucontrol build x y block rotate config // this command block on many servers, ' x ' ' y ' of building, ' block ' ( @surge-wall ) ' rotate ' 0 1 2 3 , ' config ' ( if build sorter you can write @copper, and when unit builds a sorter it will be set to copper )
  • ucontrol getBlock x y type build floor // checking floor, type and block at x y coordinates ( unit need be within 27 tiles or their radius if larger to coordinates! )
  • ucontrol within x y radius out // checking unit around radius, output 0 / 1 if unit out / in of radius
  • unbind // unbind units. Returns unit to default AI.

  • ulocate building core true @null outx outy found build // core = type of building what you search ( can be only core , storage , generator , turret , factory , repair , battery , reactor ), true / false - searching enemy building or no, then just write @null, outx / outy = returned coordinates of it , found - 0 / 1 ( found or not ), build = name of this found building
  • spawn // searching enemy spawn
  • ulocate spawn null null @null outx outy found build // output x y of spawn, found or not, build ( name )
  • damage // searching damaged block
  • ulocate damaged null null @null outx outy found build // output x y of spawn, found or not, build ( name )

A loop-timer and simple timer


How to do timer for processor without stopping it by command wait


How we know we can use a operation for this, like this code:

op add num num 1
jump l1 greaterThan num 10
// some code what processor will do if num less than 10
end
l1:
print "hi"
printflush message1
set num 0

This example good when you dont needed accurate time, in fact it is just a loop do..while


But if you need a timer which one will work with an accuracy of milliseconds and at the same time not use the wait command( since it completely stops the processor for a given time ) you can use @time and one trick

Lets do a timer what will add 1 to num every 5 second ( dont forgot, a @time counting in millisecond so 5 second = 5000

jump l1 equal loop 1
op add timeOut @time 5000
set loop 1
l1:
jump l2 greaterThanEq @time timeOut
// here you can write any code, its will be work while timer also work
print num
printflush message1
end
l2:
op add num num 1
set loop 0

First we add a 5 second to @time and named it timeOut, then set loop = 1 ( because it will always add a 5 seconds )
Then we chack if a @time >= then timeOut, if its true so 5 seconds have passed and num will +1 and loop = 0 and the timer is updated again
