Some basic about processor


  1. A processors start at line 0 (line is used instead of instruction)

  2. M-log special @ variables:

    • @thisx / @thisy
      // cooordinated x / y of this proccesor

    • @this
      // object of proccesor

    • @unit
      // variable to access a unit returned from using 'bind'. Can be used as object to sensor data from the unit.

    • @counter
      // indicates the line of code currently being performed. It's a very interesting variable, explained further here.

    • @mapw / @maph
      // map weight / height in tiles

    • @links
      // total number of blocks linked (connected) to the processor. To link a block, select the processor and then select a different block. It will have a purple diamond and a name to indicate that it is linked.

    • @ipt
      // instruction per tick (number of lines of code performed each tick. A tick is similar to FPS for most purposes)

  3. When code passes the last line, it will start again from line 0

  4. Each processor type has different processing speeds (you can read this from @ipt): micro processor(2ipt or 120 lines/sec at 60 FPS) , logic processor (480 / sec) , hyper proccesor ( 1500 / sec )

  5. A memory cell can store a limited amount of numbers (double or float32). A memory cell stores 64 numbers, a memory bank 512 numbers.

  6. All commands and words are case sensitive e.g. '@firstItem ', not '@firstitem '

  7. Most comparisons are coercive - 'true' and 'false' will also be equal to 1 and 0 respectively. Be careful, null is also coerced to 0. To avoid this, use the strict equals comparison, ===.

  8. If you write code outside Mindustry (e.g. on site or in VScode ) you can use jump labels (absolute) instead of jump numbers (relative) to avoid maintaining jumps

Input and Output

We have 4 commands of this category: read , write , draw , print



Command ' read '
return a variable from mem-cell block

syntax of read: read var cell1 num

' var ' is the name of the variable to store the output from the cell, 'cell1' is a linked mem-cell (although it can be linked in other ways), ' num ' is a 0-based index in mem-cell where to read the value from.

Example:


read aCountOFunits cell1 0
// so we created a variable 'aCountOfUnits' then we tell the processor from which cell to find a value, and index of the cell for the specific number.


Command ' write '
write a any variable to mem-cell, but it can write only numbers, variable values ( in number or 1 / 0 ( true / false )!

syntax of write: write var cell1 num

' var ' is the name of the variable where a number is stored. It can also be a static number. ' cell1 ' its a linked mem-cell, ' num ' is a 0-based index in mem-cell where to write the value.

Example:


write aCountOFunits cell1 0
// so we write a variable 'aCountOfUnits' at index 0 in mem-cell

Now lets connect a read and write commands in code!

proccessor 1:
set aCountOFunits 5
write aCountOFunits cell1 3
 proccessor 2:
read count cell1 3

Processor 1 sets a value to 'aCountOfUnits' variable and then stores this to mem-cell at index 3. Processor 2 reads a 'aCountOfUnits' at index 3, but stores it a variable named 'count'.



Command ' draw ' draw lines, shapes and images to display in a logic display. This is a detailed topic and will be completed at a later date.

syntax's of draw:

  • draw clear 0 0 0
  • draw color 0 0 0 0
  • draw col 0
  • draw stroke 0
  • draw line 0 0 0 0
  • draw rect 0 0 0 0
  • draw lineRect 0 0 0 0
  • draw poly 0 0 0 0 0
  • draw triangle 0 0 0 0 0 0
  • draw image 0 0 0

Extend of 0 you need write a name's / coordinates / icons and any information for draw any what you want.



Command ' print '
stores (appends) a string to be sent to a message block. You need to use print flush to send the stored print string to a message block.

A syntax of print: print ""

for write text you need add any text in " " , if you want write a value of any variable you need write it name and without "

Example:


print "count"
// will write a text in " " ( count )
print count
// will write a value of variable " count "

print "\n"
// will print some text from new line
print "[red]red , [#669980]some color , []white"
// you can write colored text by simple names of color [red] or by #hex colors, [] deleting color
